# AI Visibility Checklist

A practical checklist for helping AI systems accurately identify,
understand, and describe a business or product.

Created by [Daizie.ai](https://daizie.ai), a Practical Informatics
product built and calibrated by Marty Koepke.

## What this checklist covers

- Entity identity and name consistency
- Product and service descriptions
- Audience and use cases
- Pricing and availability
- Evidence supporting important claims
- Documentation, FAQs, and help content
- Reviews and independent discussion
- Structured data and crawler access
- Limitations, alternatives, and comparisons
- Consistency across public sources

## Important limitation

Completing this checklist does not guarantee inclusion, ranking, citation,
or favorable treatment by any AI system.

It helps make information clearer, more consistent, and easier to support
with available evidence.

## The checklist

### 1. Establish a clear identity

- [ ] Use the same business or product name consistently.
- [ ] State what the business or product is in one plain-English sentence.
- [ ] Identify the official website and canonical domain.
- [ ] Distinguish the business from similarly named entities.
- [ ] Provide an accurate About page.
- [ ] Name the people or organization responsible for the product.

### 2. Explain the offering

- [ ] Maintain a dedicated page for each major product or service.
- [ ] Describe the intended audience.
- [ ] Explain the primary use cases.
- [ ] List important features and capabilities.
- [ ] State meaningful limitations.
- [ ] Avoid unsupported superlatives such as “best” or “leading.”

### 3. Make pricing understandable

- [ ] Publish current pricing or explain how pricing is determined.
- [ ] Distinguish free, trial, subscription, and one-time options.
- [ ] Explain what each plan includes.
- [ ] Identify important exclusions or additional charges.
- [ ] Add the date pricing was last reviewed.

### 4. Provide supporting evidence

- [ ] Link important claims to relevant evidence.
- [ ] Separate company claims from independent evidence.
- [ ] Include dates and source names.
- [ ] Preserve the limitations and context of statistics.
- [ ] Remove outdated or unverifiable claims.

### 5. Answer real questions

- [ ] Publish an FAQ based on actual customer questions.
- [ ] Maintain useful documentation or help content.
- [ ] Explain how the product works.
- [ ] Address privacy, security, cancellation, and support.
- [ ] Explain common misunderstandings.

### 6. Establish independent presence

- [ ] Maintain accurate profiles on relevant directories.
- [ ] Invite real customers to leave honest reviews.
- [ ] Correct inaccurate directory information.
- [ ] Participate constructively in relevant communities.
- [ ] Never purchase, fabricate, or manipulate reviews.

### 7. Support technical discovery

- [ ] Provide a sitemap.
- [ ] Avoid accidentally blocking important public pages.
- [ ] Use relevant structured data accurately.
- [ ] Set canonical URLs.
- [ ] Use descriptive page titles and headings.
- [ ] Keep important information available as readable HTML.
